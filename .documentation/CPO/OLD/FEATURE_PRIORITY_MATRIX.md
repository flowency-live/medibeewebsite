# PRIORITY FRAMEWORK

## MoSCoW Categories

| Category | Definition | Target |
|----------|------------|--------|
| **Must Have** | Platform cannot launch without | Phase 0-1 |
| **Should Have** | High value, needed soon | Phase 2 |
| **Could Have** | Nice to have, can defer | Phase 3 |
| **Won't Have** | Out of scope for now | Future |

## Value Scoring (1-5)

| Score | Meaning |
|-------|---------|
| 5 | Critical to value proposition |
| 4 | High impact on adoption/revenue |
| 3 | Moderate impact |
| 2 | Low impact |
| 1 | Nice to have |

## Effort Scoring (1-5)

| Score | Meaning |
|-------|---------|
| 5 | >2 weeks, multiple engineers |
| 4 | 1-2 weeks |
| 3 | 3-5 days |
| 2 | 1-2 days |
| 1 | <1 day |

## Priority Score

**Priority = Value / Effort** (higher is better)
